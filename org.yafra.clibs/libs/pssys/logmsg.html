Generic message logging facility

Brief Description

The logmsg routines are application-independent error logging routines. The messages may be written to a specified file, to stderr, or to the syslog daemon. They are almost identical to the routines in appendix B of Stevens' Advanced Programming in the UNIX Environment. Functions:
  • void setlogfile(void) -- set name of logging file. No default.
  • void setforeground(void) -- process will run in foreground (i.e., not a daemon process).
  • void log_open(char *ident, int option, int facility) -- opens the log file for logging. See usage notes below.
The following functions all use the varargs (printf) argument format.
  • void log_ret(const char *fmt, ...) -- log a non-fatal system error and return to caller.
  • void log_sys(const char *fmt, ...) -- log a fatal system error and terminate the application.
  • void log_msg(const char *fmt, ...) -- log a non-fatal error unrelated to a system call.
  • void log_quit(const char *fmt, ...) -- log a fatal error unrelated to a system call and terminate the application.
Header file is logmsg.h. File are located in softech/libs/pssys
